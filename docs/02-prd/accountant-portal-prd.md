# Accountant Portal PRD

> Module: Accountant Portal | Version: 1.0 | Updated: 2026-06-13

---

## Overview

The accountant portal is the multi-client management interface for tax professionals. It is accessed at `/accountant` after login with role=accountant. It provides a KPI dashboard, a client/engagement list, engagement workspaces (see `engagement-workspace-prd.md`), the AI deduction scanner, and firm management tools.

---

## Key Screens

### 1. Accountant Dashboard (`/accountant`)

A KPI-focused overview of the accountant's full client portfolio.

**KPI tiles (top row):**
- Active engagements (count, by status breakdown)
- ReadyToFile count (ready right now)
- Missing docs (count of engagements with ≥1 mandatory doc not yet uploaded)
- Average readiness score (across all active engagements)
- Overdue items (checklist items past their due date)

**Next-actions panel:**
- AI-generated list of top-priority actions across all clients
- Each row: colored priority dot + client name + issue (e.g., "Jaaropgaaf missing") + action link
- Sorted by: required > recommended, by deadline proximity
- Refresh button (re-runs AI generation)

**Recent activity feed:**
- Last 24 hours of events: document uploads, checklist status changes, messages
- Each event: timestamp, client name, event description, link

**Behavior:**
- FR-ACT-001: Dashboard data loads within 3 seconds (cached scores, no live DB aggregation on load)
- FR-ACT-002: Next-actions panel shows max 10 rows — "View all" links to the AI actions list
- FR-ACT-003: KPI tiles update every 60 seconds via polling
- FR-ACT-004: Recent activity feed shows events across ALL of this accountant's clients (not just one)

### 2. Client List (`/accountant/clients`)

A searchable, filterable list of all clients managed by this accountant.

**List columns:**
- Client name, email
- Tax year
- Engagement status badge
- Readiness score (ProgressRing mini)
- Open tasks count
- Last activity date
- Quick-action: Send reminder / Open engagement

**Filters:**
- By engagement status (multi-select)
- By readiness score range (e.g., <50, 50–75, 75+)
- By tax year
- By missing doc flag
- Search by name or email

**Behavior:**
- FR-ACT-005: Default sort: ReadyToFile first, then by readiness score descending
- FR-ACT-006: Clicking a row opens the engagement workspace for that client
- FR-ACT-007: "New client" button opens the invitation flow
- FR-ACT-008: Bulk action: "Send reminders" sends email to all selected clients with open tasks

### 3. Invitation Flow (`/accountant/invite`)

How accountants onboard new clients.

**Steps:**
1. Enter client email address
2. Select tax year
3. Optionally: pre-fill client name
4. System sends invitation email to client
5. Client accepts → engagement auto-created + checklist auto-generated

**Behavior:**
- FR-ACT-009: Invitation email sent within 30 seconds of accountant clicking "Send"
- FR-ACT-010: Invitation expires after 7 days — accountant can resend
- FR-ACT-011: If client already has an account, they see the invitation in their portal tasks
- FR-ACT-012: If client is new, invitation email contains registration link with invitation token pre-filled
- FR-ACT-013: Accepting the invitation automatically links the client to this accountant's firm and creates a Draft engagement

### 4. AI Actions Center (`/accountant/actions`)

A full-page view of AI-generated next actions across all clients.

**Columns:** Priority | Client | Tax Year | Issue | Recommended Action | Status | Last Updated

**Statuses:** Open / Acknowledged / Done / Dismissed

**Filters:** by priority, by client, by issue category (document, checklist, risk, deadline)

**Behavior:**
- FR-ACT-014: Actions are generated by the AccountantAction model + AI engine
- FR-ACT-015: Marking an action "Done" writes to PortalAuditLog
- FR-ACT-016: Dismissed actions do not re-appear unless the underlying state changes
- FR-ACT-017: "Re-generate actions" button triggers fresh AI analysis for all clients

### 5. Engagement Workspace (`/accountant/engagements/{id}`)

See [engagement-workspace-prd.md](engagement-workspace-prd.md) for the full specification.

This is the per-client, per-tax-year workspace. All 7 tabs (Overview, Checklist, Documents, Income, Expenses, Risks, Audit).

### 6. Firm Management (`/accountant/firm`)

Visible to `firm_manager` and `admin` roles only.

**Sub-tabs:**
- **Team** — list of accountants in the firm. Add/remove/change role.
- **Clients** — list of all clients across all accountants in the firm. Reassign engagements.
- **Settings** — firm name, logo, contact email, subscription plan.
- **Billing** (future) — usage summary, invoices.

**Behavior:**
- FR-ACT-018: Firm manager can reassign an engagement from accountant A to accountant B
- FR-ACT-019: Adding a new accountant sends an invitation email to their address
- FR-ACT-020: Removing an accountant from the firm does NOT delete their clients' engagements (must be reassigned first)

### 7. Rule Management (`/accountant/rules`)

Admin-visible interface for managing the tax rule database. See also `rule-engine-prd.md`.

**In scope for accountant portal:**
- View the 28 active tax rules with their current values
- Filter by topic, category, user_type
- View rule history (who changed what and when)
- Flag a rule for review ("I think this value is wrong")

**Out of scope for accountant portal:**
- Creating new rules (Admin/super_admin only)
- Approving rule changes (Admin/super_admin only)

## Notification Preferences (Accountant-Specific)

| Notification | Default | Channel |
|-------------|---------|---------|
| Client uploads document | Email + Push | On |
| Client sends message | Email + Push | On |
| Engagement reaches ReadyToFile | Email | On |
| Overdue checklist item | Email | Daily digest |
| AI actions summary | Email | Weekly |
| New client invitation accepted | Email + Push | On |

**Behavior:**
- FR-ACT-021: Accountant can customize notification preferences per event type in Settings
- FR-ACT-022: "Daily digest" mode batches all events into a single 8:00am email

---

## Multi-Tenant Isolation Rules

1. An accountant sees ONLY clients where `engagement.accountant_user == request.user`
2. A firm_manager sees all clients where `engagement.accountant_user.firm == request.user.firm`
3. An admin sees all clients (all firms)
4. No cross-firm data leakage is possible at the application or database level
